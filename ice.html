ICE Challenges

In-class exercises completed throughout WDD 130. Each entry includes a description and a link to the live page.

ICE 01 — HTML Basics

This exercise introduced the foundational structure of an HTML document. We practiced writing proper doctype declarations, setting up head and body sections, and using basic tags like headings, paragraphs, and links. It was the first time writing HTML by hand rather than using a template.

What I learned: How the browser interprets HTML structure, why semantic tags matter, and how to properly nest elements.

View ICE 01 →

ICE 02 — CSS Selectors

This challenge focused on writing CSS rules to style HTML elements. We practiced targeting elements by tag, class, and ID, and explored how specificity determines which styles apply when rules conflict.

What I learned: The cascade and specificity hierarchy, how to write clean selectors, and the difference between class and ID usage.

View ICE 02 →

ICE 03 — CSS Box Model

We dove into the box model — understanding how margin, border, padding, and content area interact. We also practiced using box-sizing: border-box to simplify layout math.

What I learned: Why elements take up the space they do, how to control spacing precisely, and when to use margin vs padding.

View ICE 03 →

ICE 04 — CSS Grid Layout

This exercise applied CSS Grid to build a multi-column page layout. We used grid-template-columns, grid-column placement, and let the browser create implicit rows as needed.

What I learned: How to define a grid, place items explicitly using column and row lines, and when to let the browser handle placement automatically.

View ICE 04 →

ICE 05 — Flexbox Navigation

We built a responsive navigation bar using Flexbox. The exercise covered justify-content, align-items, and how flex items distribute space along the main axis.

What I learned: The difference between Flexbox and Grid, how to space nav links evenly, and how flex-direction changes layouts on smaller screens.

View ICE 05 →

ICE 06 — Responsive Design

This final ICE challenge introduced media queries to make a page look good across mobile, tablet, and desktop screen sizes. We followed a mobile-first approach and overrode styles at wider breakpoints.

What I learned: How to write @media rules, why mobile-first is the recommended approach, and how to test responsiveness using browser dev tools.
